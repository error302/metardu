# METARDU Mobile App — Production Build Guide

This document covers building the METARDU PWA as a native Android app using Capacitor.

## Architecture

METARDU is a **Next.js 14 PWA** that wraps as a native Android app via [Capacitor](https://capacitorjs.com/). The same codebase powers:

1. **Web app** (default) — `next build` with `output: 'standalone'`
2. **Mobile app** — `MOBILE_BUILD=true next build` with `output: 'export'` (static HTML/JS/CSS), then `npx cap sync android` copies the static assets into the native project

The mobile app has full access to native device features via Capacitor plugins:

| Plugin | Purpose |
|--------|---------|
| `@capacitor/geolocation` | GPS positioning (more accurate than browser Geolocation API) |
| `@capacitor-community/bluetooth-le` | BLE connection to total stations and GNSS rovers |
| `@capacitor/filesystem` | Offline data storage (IndexedDB fallback for web) |
| `@capacitor/file-picker` | Import CSV/GeoJSON/DXF files from device storage |
| `@capawesome/capacitor-live-update` | OTA updates (skip Play Store review for bug fixes) |

## First-time setup

```bash
# 1. Install dependencies (already in package.json)
npm install --legacy-peer-deps

# 2. Initialize the Android project (creates android/ directory)
node scripts/mobile-setup.mjs

# 3. Open in Android Studio (to install SDK + build tools)
npm run mobile:open
```

Android Studio will prompt you to install the required SDK components. Accept all defaults.

## Building the APK

### Debug APK (for testing)

```bash
npm run mobile:build
# Output: android/app/build/outputs/apk/debug/app-debug.apk
```

### Release APK (for production)

1. Generate a signed upload key (one-time):
   ```bash
   keytool -genkey -v -keystore metardu-release.keystore -alias metardu -keyalg RSA -keysize 2048 -validity 10000
   ```
   Store this keystore file securely — losing it means you can't update the app on Play Store.

2. Configure signing in `android/app/build.gradle`:
   ```gradle
   signingConfigs {
       release {
           storeFile file('../../metardu-release.keystore')
           storePassword System.getenv('METARDU_KEYSTORE_PASSWORD')
           keyAlias 'metardu'
           keyPassword System.getenv('METARDU_KEY_PASSWORD')
       }
   }
   buildTypes {
       release {
           signingConfig signingConfigs.release
           minifyEnabled true
           proguardFiles getDefaultProguardFile('proguard-android-optimize.txt'), 'proguard-rules.pro'
       }
   }
   ```

3. Build the release APK:
   ```bash
   cd android
   ./gradlew assembleRelease
   # Output: android/app/build/outputs/apk/release/app-release.apk
   ```

### App Bundle (for Play Store)

```bash
cd android
./gradlew bundleRelease
# Output: android/app/build/outputs/bundle/release/app-release.aab
```

Play Store requires `.aab` format (not `.apk`).

## After every web code change

```bash
# Rebuild static export + sync to native project
node scripts/mobile-setup.mjs --sync

# Or manually:
MOBILE_BUILD=true npx next build
npx cap sync android

# Then rebuild the APK:
cd android && ./gradlew assembleDebug
```

## Mobile-specific UI considerations

The METARDU UI is designed mobile-first:

| Element | Mobile behavior |
|---------|----------------|
| **Navigation** | Bottom tab bar (`MobileNav`) on screens < `md` (768px), top navbar on larger |
| **Map page** | Touch-optimized controls, gesture lock to prevent accidental pan/zoom during stakeout |
| **Field book** | Full-screen observation forms with large tap targets |
| **Tables** | Horizontal scroll wrappers, never break layout |
| **Modals** | Bottom sheet style on mobile (`vaul` library) |
| **Text inputs** | 44px minimum tap target, numeric keyboards for coordinate entry |

## Required Android permissions

The app declares these permissions in `android/app/src/main/AndroidManifest.xml` (auto-generated by Capacitor):

| Permission | Why |
|------------|-----|
| `ACCESS_FINE_LOCATION` | GPS positioning for stakeout and field collection |
| `ACCESS_COARSE_LOCATION` | Fallback GPS (less accurate but faster) |
| `BLUETOOTH` | Connect to total stations via BLE (Android 11 and below) |
| `BLUETOOTH_ADMIN` | BLE scanning |
| `BLUETOOTH_CONNECT` | BLE connection (Android 12+) |
| `BLUETOOTH_SCAN` | BLE scanning (Android 12+) |
| `INTERNET` | API calls, map tiles, Sentry |
| `ACCESS_NETWORK_STATE` | Offline detection |
| `WRITE_EXTERNAL_STORAGE` | Export DXF/CSV/PDF files (scoped storage on Android 10+) |
| `READ_EXTERNAL_STORAGE` | Import files (scoped storage on Android 10+) |
| `CAMERA` | Beacon photo capture |
| `VIBRATE` | Stakeout proximity alerts |

## OTA updates (Live Update)

The app uses `@capawesome/capacitor-live-update` to ship bug fixes without going through Play Store review:

```bash
# Bundle the current web build into a live update
npx capawesome-live-update bundle

# Upload to Capawesome Cloud (or self-host)
npx capawesome-live-update publish
```

Major version bumps (new native plugins, new Android permissions) still require Play Store review.

## Troubleshooting

### "Cannot find module '@capacitor/cli'" 
Run `npm install --legacy-peer-deps` to install all dependencies.

### `npx cap add android` fails
Make sure Android Studio is installed and `ANDROID_HOME` env var is set.

### Build fails with "Minimum SDK version 23 required"
Open `android/variables.gradle` and bump `minSdkVersion` to 23 or higher.

### App crashes on launch
Check `adb logcat` for the stack trace. Most common cause: missing internet permission or wrong server URL.

### Web assets not updating
Run `npx cap sync android` after every web change. The native project caches old assets otherwise.

### BLE connection fails on Android 12+
Make sure `BLUETOOTH_CONNECT` and `BLUETOOTH_SCAN` permissions are declared and runtime-requested.

## Mobile CI/CD

The CI workflow (`.github/workflows/ci.yml`) includes a `mobile-build-check` job that runs on every PR:

```bash
MOBILE_BUILD=true npx next build
test -d out && echo "✓ static export OK"
test -f out/index.html && echo "✓ index.html present"
```

This catches static export regressions (e.g., using a server-only API in a mobile page) before they reach the APK build.

For full APK builds in CI, add a `mobile-release.yml` workflow that runs on tagged releases:
1. Build static export
2. Run `npx cap sync android`
3. Build release AAB
4. Upload AAB as a GitHub artifact
5. (Optional) Auto-publish to Play Store via `r0adkll/upload-google-play` action
