Common Sheet Structure (3 sheets per file):

=== Sheet 0: "Co-ordinates to be transformed" (158 rows × 6 cols) ===
  Row 2: Title - "TRANSFORMATION OF CASSINI CO-ORDINATES TO UTM COORDINATES BY USE OF A FOUR PARAMETER"
  Row 3: Topo sheet ID (e.g., "103/I and III", "148_2", "148_3", "148_4")
  Row 4: Headers "UTM", "DOS", "CASSINI"
  Rows 5-18: Projection parameters:
    Row 5: "parameters" header
    Row 6: flattening - Cassini: 0.003398355, UTM: 0.003407561378699334
    Row 7: eccentricity
    Row 8: semi major axis - Cassini: 20926348.0, UTM: 6378249.145
    Row 9: semi minor axis
    Row 10: spheroid - Cassini: Clarke 1858, UTM: Clarke 1880
    Row 11: projection - UTM: TM
    Row 12: unit - Cassini: Feet, UTM: metre
    Row 13: Meridian of origin - Cassini: 37E, UTM: 39E
    Row 14: Latitude of Origin - both: Equator
    Row 15: Scale Factor origin - UTM: 0.9996
    Row 16-17: False Origin - UTM: 500000W, 10000000S
    Row 18: Datum - both: New(1960) arc
  Rows 22-23: Column headers
    Col A: Station name
    Col B: Cassini Northings (feet)
    Col C: Cassini Eastings (feet)
    Col D: Conformal Cassini Eastings (computed correction applied)
    Col E: UTM Northings (metres) - blank for non-control points
    Col F: UTM Eastings (metres) - blank for non-control points
  Row 24+: Station data
    First 3 stations have UTM coordinates (control points used to derive parameters)
    Remaining ~130 stations have only Cassini coordinates (to be transformed)

  Conformal Easting formula (applied in col D):
    x_c = x + x³/(6R²) + x⁵/(24R⁴) + ...
    where R = sqrt(a*b) for the Clarke 1858 ellipsoid

=== Sheet 1: "solution of four parameters" (40 rows × 8 cols) ===
  Contains the matrix solution for the 6-parameter transformation (4+2 polynomial).
  
  Key rows:
    Rows 3-8: L vector (6×1) - UTM coordinates of 3 control points (E,N pairs)
    Rows 3-8: A matrix (6×6) - conformal Cassini coords [x, -y] + identity
    Rows 13-18: AT matrix (6×6) - transpose of A
    Rows 21-28: ATL vector (6×1) and ATA matrix (6×6)
    Rows 31-38: Solution vector (6×1) and (ATA)⁻¹ matrix (6×6)
    
  Solution Parameters (rows 33-38):
    Row 33: P (scale/rotation for eastings) ≈ 0.30483...
    Row 34: Q (scale/rotation for northings) ≈ 0.00001... (very small)
    Row 35: A (quadratic term) ≈ -2.5e-10
    Row 36: B (quadratic term) ≈ 5e-11
    Row 37: Cx (translation eastings) ≈ 277474-277484
    Row 38: Cy (translation northings) ≈ 10000196-10000208

  Transformation formula:
    UTM_E = P * conformal_cass_E + Q * cassini_N + A * E² + B * E*N + Cx
    UTM_N = P2 * conformal_cass_E + Q2 * cassini_N + A2 * E² + B2 * E*N + Cy
    (where the full 6-parameter solution applies a 2nd-degree polynomial)

  Actually the 4 parameters (P, Q, Cx, Cy) represent:
    - P: scale factor for eastings (~0.3048, which is feet-to-metres conversion)
    - Q: rotation/skew term (very small, ~0.00001)
    - Cx: origin shift in eastings (~277475 metres)
    - Cy: origin shift in northings (~10000198 metres)
  
  Plus 2 additional polynomial terms (A, B) for non-linearity correction.

=== Sheet 2: "transformed coordinates" (94 rows × 5-6 cols) ===
  Contains the transformed UTM coordinates for all stations.
  
  Headers (rows 2-3):
    Col A: Station name
    Col B: Cassini Northings (feet)
    Col C: Cassini Eastings (feet)
    Col D: Transformed UTM Northings (metres)
    Col E: Transformed UTM Eastings (metres)
    Col F: (sometimes present) Station name repeated

7. PDF FILE: TS09E_thomas_gicira_9237.pdf
================================================================================
Title: "Polynomial Transformation Of Cassini Coordinates To UTM Coordinates On The Excel Spreadsheet"
Author: T.G. Gacoki - Kenya National Highways Authority
Published: FIG Congress 2018, Istanbul, Turkey, May 6–11, 2018
Pages: 12

Content Summary:
  - Academic paper describing the exact methodology used in the XLS files
  - Explains the 12-parameter polynomial transformation (2nd degree)
  - Uses 7 common points (6 minimum) for least squares solution
  - Three interconnected Excel worksheets approach (same as XLS files)
  - Cassini→UTM conversion: conformal correction applied to Cassini eastings first
  - General formula: X = Ao + A1*x + A2*y + A3*x² + A4*xy + A5*y²
  - Includes conformal correction formula: x_c = x + x³/(6R²) + x⁵/(24R⁴)
  - Uses Clarke 1858 (Cassini) and Clarke 1880 (UTM) ellipsoids
  - Cassini origin: 37°E, Equator; UTM origin: 39°E, Equator, SF=0.9996
  - 7-parameter (Helmert) alternative mentioned using ellipsoidal coordinates

  The paper is a FIG conference proceeding that documents the spreadsheet methodology.
  It does NOT contain specific topo sheet data or parameters - it's a methodology guide.

9. PROGRAMMATIC PARSING GUIDE
================================================================================

To parse sub-sheet corners from Sheet 5 ("sheet corners"):
  1. Skip row 0 (blank), row 1 (headers)
  2. Scan column A for non-empty values → this is the topo sheet ID
  3. Scan column B for non-empty values → this is the sub-sheet number
  4. The 4 data rows following the header row are the corners:
     Row +0: Col C=Corner1_E, Col D=Corner1_N, Col E=Corner1_CasX, Col F=Corner1_CasY
     Row +1: Col C=Corner2_E, Col D=Corner2_N, Col E=Corner2_CasX, Col F=Corner2_CasY
     Row +2: Col C=Corner3_E, Col D=Corner3_N, Col E=Corner3_CasX, Col F=Corner3_CasY
     Row +3: Col C=Corner4_E, Col D=Corner4_N, Col E=Corner4_CasX, Col F=Corner4_CasY
  5. Skip the blank separator row
  6. Repeat

To parse whole-sheet corners from Sheet 3 ("SHT CASN TO UTM (2)"):
  1. Skip row 0 (headers), row 1 (blank)
  2. Scan column A for non-empty values → topo sheet ID
  3. Next 4 rows: 4 corners
     Col B=Cassini_X, Col C=Cassini_Y, Col D=UTM_E, Col E=UTM_N
  4. Skip separator row (contains commas)
  5. Repeat

To parse whole-sheet corners from Sheet 6 ("SHT UTM TO CASSIN"):
  1. Same as Sheet 3 but columns swapped:
     Col B=UTM_E, Col C=UTM_N, Col D=Cassini_X, Col E=Cassini_Y

To parse individual XLS files:
  1. Sheet 0: Get control points (rows 24-26, columns E,F have UTM coords)
  2. Sheet 1: Get parameters from rows 33-38 (P,Q,A,B,Cx,Cy)
  3. Sheet 2: Get transformed coordinates (rows 4+)
